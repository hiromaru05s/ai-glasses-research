# ② AIグラス市場シンクタンク — 構築ガイド（AWS統一版）

## 全体アーキテクチャ

```
EventBridge (cron: 毎朝7:00 JST)
    ↓ トリガー
ECS Fargate タスク起動
    ↓
Docker コンテナ内で Claude Code ヘッドレス実行
    ↓
Web検索 / YouTube / X / Reddit から情報収集
    ↓
日次レポート (.md) 生成
    ↓
Google Drive に自動アップロード
（＋ S3 にバックアップ保存）
    ↓
CloudWatch Logs にログ記録
```

---

## 選択肢: EC2 vs ECS Fargate

| | EC2 (t3.medium) | ECS Fargate |
|---|---|---|
| **月額インフラ** | ~$30（常時起動）/ ~$10（スポット） | ~$3〜5（実行時間分のみ課金） |
| **管理** | サーバー管理必要 | フルマネージド |
| **スケール** | 手動 | 自動 |
| **他組織追加** | 同じEC2に同居（安い） | 別タスクとして追加（簡単） |
| **おすすめ** | 全4組織を本格稼働する段階 | まず②だけ動かす段階 |

**推奨**: まずFargateで②を動かす → 組織が増えたらEC2に統合

## 構築手順

### Step 0: 前提条件

```bash
# AWS CLI インストール・設定済み
aws configure
# → AWS Access Key, Secret Key, Region (ap-northeast-1) を設定

# Docker インストール済み
docker --version
```

---

### Step 1: ローカルでDocker動作確認

```bash
# リポジトリのルートに移動
cd 組織構築/

# .env を作成
cp .env.example .env
# → .env を編集して ANTHROPIC_API_KEY を設定

# ビルド
docker compose build glass-market-daily

# ローカル実行テスト
docker compose run --rm glass-market-daily
```

→ `reports/daily/YYYY-MM-DD.md` が生成されれば成功。

**ここで必ずレポートの質を確認する。** CLAUDE.mdやSKILL.mdの調整はこの段階で行う。

---

### Step 2: ECR にDockerイメージをpush

```bash
# ECRリポジトリ作成
aws ecr create-repository \
  --repository-name glass-market-daily \
  --region ap-northeast-1

# ECRにログイン
aws ecr get-login-password --region ap-northeast-1 | \
  docker login --username AWS --password-stdin \
  ACCOUNT_ID.dkr.ecr.ap-northeast-1.amazonaws.com

# イメージをビルド & push
docker build -t glass-market-daily -f org-glass-market/infra/docker/Dockerfile org-glass-market/
docker tag glass-market-daily:latest ACCOUNT_ID.dkr.ecr.ap-northeast-1.amazonaws.com/glass-market-daily:latest
docker push ACCOUNT_ID.dkr.ecr.ap-northeast-1.amazonaws.com/glass-market-daily:latest
```

---

### Step 3: Secrets Manager にAPIキーを格納

```bash
# Anthropic APIキー
aws secretsmanager create-secret \
  --name anthropic-api-key \
  --secret-string "sk-ant-xxxxxxxx"

# Google Drive サービスアカウント（オプション）
aws secretsmanager create-secret \
  --name gdrive-service-account \
  --secret-string file://path/to/service-account.json

# Google Drive フォルダID（オプション）
aws secretsmanager create-secret \
  --name gdrive-folder-id \
  --secret-string "YOUR_FOLDER_ID"
```

---

### Step 4: ECS クラスター & タスク定義を作成

```bash
# クラスター作成
aws ecs create-cluster --cluster-name ai-org-cluster

# CloudWatch ロググループ作成
aws logs create-log-group --log-group-name /ecs/glass-market-daily

# IAMロール作成（初回のみ）
# ecsTaskExecutionRole: ECRからイメージpull、Secrets Managerアクセス
# ecsTaskRole: S3アクセス等（必要なら）

# タスク定義を登録
# infra/aws/ecs-task-definition.json の ACCOUNT_ID を自分のIDに置換してから:
aws ecs register-task-definition \
  --cli-input-json file://org-glass-market/infra/aws/ecs-task-definition.json
```

---

### Step 5: EventBridge でスケジュール設定

```bash
# EventBridge ルール作成（毎朝7:00 JST = 22:00 UTC）
aws events put-rule \
  --name glass-market-daily-schedule \
  --schedule-expression "cron(0 22 * * ? *)" \
  --state ENABLED \
  --description "毎朝7:00 JSTにAIグラス市場日次調査を実行"

# ECSタスクをターゲットに設定
aws events put-targets \
  --rule glass-market-daily-schedule \
  --targets '[{
    "Id": "glass-market-daily",
    "Arn": "arn:aws:ecs:ap-northeast-1:ACCOUNT_ID:cluster/ai-org-cluster",
    "RoleArn": "arn:aws:iam::ACCOUNT_ID:role/ecsEventsRole",
    "EcsParameters": {
      "TaskDefinitionArn": "arn:aws:ecs:ap-northeast-1:ACCOUNT_ID:task-definition/glass-market-daily",
      "TaskCount": 1,
      "LaunchType": "FARGATE",
      "NetworkConfiguration": {
        "awsvpcConfiguration": {
          "Subnets": ["subnet-xxxxxxxx"],
          "SecurityGroups": ["sg-xxxxxxxx"],
          "AssignPublicIp": "ENABLED"
        }
      }
    }
  }]'
```

---

### Step 6: 手動テスト実行

```bash
# ECSタスクを手動起動
aws ecs run-task \
  --cluster ai-org-cluster \
  --task-definition glass-market-daily \
  --launch-type FARGATE \
  --network-configuration '{
    "awsvpcConfiguration": {
      "subnets": ["subnet-xxxxxxxx"],
      "securityGroups": ["sg-xxxxxxxx"],
      "assignPublicIp": "ENABLED"
    }
  }'

# ログ確認
aws logs tail /ecs/glass-market-daily --follow
```

---

### Step 7: Google Drive連携（オプション）

Step 3でSecretsに登録済みなら自動でアップロードされる。
未設定の場合は `構築ガイド（前版）` のStep 4を参照。

## 将来の組織追加時の手順

新しい組織を追加する場合:

1. `org-新組織名/` ディレクトリを作成
2. `CLAUDE.md`, `SKILL.md` を設計
3. `Dockerfile` + `entrypoint.sh` を作成（テンプレートをコピー）
4. `docker-compose.yml` にサービス追加
5. ローカルでテスト
6. ECRにpush → ECSタスク定義登録 → EventBridgeルール追加

基本的に**②のコピー＆カスタマイズ**で新組織が立ち上がる。
