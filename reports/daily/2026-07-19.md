## 技術・開発者向け情報（新規のみ）

### SDK・API更新

- **[🟡] Android XR の `XR Runtime` が初の beta に到達** — Jetpack のランタイム基盤 `androidx.xr.runtime` が `1.0.0-beta01` として公開された。
  - **事実**: 2026年7月15日付の `XR Runtime | Jetpack | Android Developers` で `1.0.0-beta01` が公開され、`Session.create` の suspend 化、`SessionTestRule` と `FakePerceptionManager` の追加、`androidx.xr.arcore` とのフレーム/変換連携強化が案内された。
  - **推測 / 含意**: Android XR のセッション初期化やテスト基盤は実験段階から安定化フェーズへ移りつつある。社内PoCが独自モックや旧初期化フローに依存しているなら、今のうちに差し替え対象を洗い出した方がよい。
  - 📎 [XR Runtime | Jetpack | Android Developers](https://developer.android.com/jetpack/androidx/releases/xr-runtime) — Android Developers

- **[🟡] `ARCore for Jetpack XR` も `beta01` に進んだ** — Jetpack XR 向けの ARCore 連携層がベータへ進み、型整理が進んだ。
  - **事実**: 2026年7月15日付の `XR ARCore | Jetpack | Android Developers` で `1.0.0-beta01` が公開され、`Anchor` `Plane` `Session` `TrackingState` `AugmentedImages` `AugmentedObject` `Config` などが `AnchorSpace` 前提へ寄せられたことが記載された。
  - **推測 / 含意**: Scene 理解やアンカー管理を使う Android XR 実装は、`SceneCore beta01` と合わせて型名や追跡状態の扱いをまとめて更新する方が安全で、ライブラリを個別追従するより一括棚卸しの方がコストを抑えやすい。
  - 📎 [XR ARCore | Jetpack | Android Developers](https://developer.android.com/jetpack/androidx/releases/xr-arcore) — Android Developers

- **[⚪] `XR Compose` が `alpha16` で API 整理を継続** — 空間UI向け Compose 層も更新された。
  - **事実**: 2026年7月15日付の `XR Compose | Jetpack | Android Developers` で `1.0.0-alpha16` が公開され、`ComponentOverride` / `OverrideEnablement` / `SpatialDialogProperties` の削除と、空間オーディオ出力デバイス API の追加が案内された。
  - **推測 / 含意**: Compose ベースで spatial UI を試しているなら、互換レイヤー温存より API 追従を優先した方が後工程の破綻を避けやすい。特に音声出力先を扱う体験は早めに検証価値がある。
  - 📎 [XR Compose | Jetpack | Android Developers](https://developer.android.com/jetpack/androidx/releases/xr-compose) — Android Developers
