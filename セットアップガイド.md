## Step 1: GitHubリポジトリ作成

1. https://github.com/new → Private で作成
2. SSH鍵を設定（未設定の場合）:
```powershell
ssh-keygen -t ed25519 -C "your-email@example.com"
cat ~/.ssh/id_ed25519.pub
# → GitHubのSSH keys設定に登録
```

## Step 3: Automationをローカルに展開

リポジトリを clone したら、追跡済み automation 定義を Codex ローカル環境へ展開する:

```bash
cd /path/to/ai-glasses-research
bash automations/install-glass-market-daily.sh
```

これで以下が自動生成される:

- `$CODEX_HOME/automations/glass-market-daily/automation.toml`
- `$CODEX_HOME/automations/glass-market-daily/memory.md`

`automation.toml` の `cwd` には、その Mac 上のこのリポジトリ実パスが自動で入る。

この repo には `.codex/config.toml` も含まれており、Codex の sandbox を `workspace-write + network_access = true` に寄せている。これにより日次 automation が `git pull` / `git push` と Web 調査を実行できる。

Codex を先に開いたまま clone / pull した場合は、この設定を確実に読ませるために一度だけ Codex を再起動する。

## Step 4: 初回実行

サイドバー「Scheduled」から:
- `glass-market-daily` → 「Run now」
- 権限承認ダイアログが出るので全て承認

一度承認すれば以降の自動実行では承認不要。
